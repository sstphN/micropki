# MicroPKI

Минималистичная инфраструктура публичных ключей (PKI) на языке Go. Проект разработан как учебное пособие по реализации криптографических протоколов и управления сертификатами.

## Текущий статус (Sprint 1)

Реализована базовая инициализация корневого удостоверяющего центра (Root CA):
- **Генерация ключей:** RSA (4096 бит) или ECC (P-384).
- **Безопасность:** Хранение закрытых ключей в формате PKCS#8 с шифрованием AES-256.
- **Сертификация:** Генерация самоподписанных сертификатов X.509 v3.
- **Политики:** Автоматическое создание документа политики безопасности (`policy.txt`).
- **Логирование:** Структурированный вывод всех операций в реальном времени.

Установка
Для сборки требуется Go версии 1.21 или выше.
```
go mod tidy
go build -o micropki ./cmd/micropki/
```
Использование
Инициализация Root CA

# Создайте файл с паролем для ключа
```
echo "your_secure_password" > pass.txt
```

# Инициализация CA
```
./micropki ca init \
    --subject "/CN=MicroPKI Root CA/O=Organization/C=RU" \
    --passphrase-file pass.txt \
    --out-dir ./pki \
    --key-type rsa \
    --validity-days 3650
```    


Параметры команды ca init

Аргумент,Описание,Пример
--subject,Имя владельца (DN),/CN=My Root CA
--key-type,Тип ключа (rsa / ecc),rsa
--passphrase-file,Файл с паролем для ключа,./pass.txt
--out-dir,Директория вывода,./pki
--validity-days,Срок действия в днях,3650


Тестирование командой:
```
go test -v ./internal/...
```

Проверка сертификата (OpenSSL)
```
# Проверка данных сертификата
openssl x509 -in ./pki/certs/ca.cert.pem -text -noout

# Проверка приватного ключа
openssl pkey -in ./pki/private/ca.key.pem -text -noout
```
